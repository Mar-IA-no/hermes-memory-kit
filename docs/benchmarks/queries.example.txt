# Embedding benchmark query set — example template
#
# FORMAT:
#   <query text>               (no annotation; retrieval scored only on auto metrics)
#   <query text> | expected:   (negative control — should return null_retrieval)
#   <query text> | expected: 17,42,88   (expected relevant chapter IDs for Precision@k / Recall@k)
#
# Lines starting with # are comments.
#
# BEST PRACTICE:
# - Include a mix of precise queries (specific nouns or proper nouns) and
#   broad topical queries. Annotate `expected:` for the ones you know the
#   ground truth on — those drive the strongest verdict signal.
# - Include 1-2 negative controls (unrelated topics) to detect false-positive
#   retrieval differences between providers.
# - Keep the query set small (8-15 queries is plenty). More queries =
#   better stats but longer runs and higher API cost.
# - DO NOT put sensitive corpus content in query text if you plan to share
#   the queries file publicly. Keep your real queries in a local file
#   outside any repo (e.g. ~/hmk-benchmarks/queries.txt).
